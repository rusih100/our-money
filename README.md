# Our Money — CSV Annotation Desktop App

Полированное, «клавиатура-first» десктоп-приложение для ручной разметки
банковских CSV-выписок. Загружаете экспорт из банка, быстро классифицируете
каждую транзакцию хоткеями и выгружаете обогащённый CSV с новыми колонками.

Построено на **Tauri 2** (ядро на Rust) + **React + TypeScript + Vite**,
стилизация — **Tailwind CSS v4**, парсинг — **PapaParse**, состояние —
**Zustand**. Тёмная тема по умолчанию, переключаемая на светлую.

---

## Возможности

- **Фокус-режим** — карточка одной транзакции: сумма с цветовой кодировкой
  (красный расход / зелёный доход), описание, банковская категория, дата, MCC.
- **Разметка хоткеями** — категория по позиции (`1`–`9`), recurring, need/want/
  saving, заметка. Мышь полностью опциональна.
- **Таблица/обзор** — виртуализированный список (300+ строк плавно) с фильтрами
  и живой аналитической сводкой (по need/want, повторяемости, категориям).
- **Управление категориями** — добавление/переименование/сортировка/удаление;
  порядок задаёт хоткеи 1–9. Сид из существующих категорий датасета.
- **Автосохранение прогресса** — привязано к файлу, возобновляет ровно с того
  места, где остановились.
- **Экспорт** — новый CSV с тем же разделителем/кодировкой + 5 новых колонок.
  Исходный файл никогда не перезаписывается.

## Горячие клавиши

| Клавиша | Действие |
|---|---|
| `j` / `↓` | следующая транзакция |
| `k` / `↑` | предыдущая транзакция |
| `1`…`9` | назначить категорию по её позиции в списке |
| `r` | переключить «повторяющийся» (is_recurring) |
| `n` | пометить **need** |
| `w` | пометить **want** |
| `s` | пометить **saving** |
| `c` | фокус на поле заметки (`Esc` — назад в режим клавиш) |
| `Enter` | подтвердить строку и перейти к следующей (annotated = true) |
| `u` | отменить последнее действие разметки |
| `/` | поиск/фильтр по описанию |
| `g` `g` | перейти к первой неразмеченной строке |
| `?` | показать/скрыть шпаргалку клавиш |
| `Ctrl`+`S` | экспорт CSV |

> **Важно:** пока курсор находится в текстовом поле (заметка/поиск), хоткеи не
> срабатывают — печатается обычный текст. `Esc` возвращает в режим клавиш.

## Схема аннотаций

На экспорте к каждой строке добавляются **5 новых колонок** (оригинальные
колонки не меняются и идут первыми):

| Колонка | Тип | Описание |
|---|---|---|
| `true_category` | строка | категория, назначенная пользователем из управляемого списка |
| `is_recurring` | bool (`true`/`false`) | повторяющийся платёж или разовый |
| `need_want` | enum: `need` \| `want` \| `saving` | классификация траты |
| `note` | строка | свободная заметка (опционально) |
| `annotated` | bool (`true`/`false`) | строка была размечена (для прогресса и резюме) |

## Формат входных данных

Банковский экспорт CSV: разделитель `;`, десятичная запятая (`,`),
поля в двойных кавычках, кодировка **Windows-1251 или UTF-8** (определяется
автоматически в Rust-ядре через `encoding_rs`).

Приложение рендерит любые присутствующие колонки. Приоритетные поля для
отображения: `Описание`, `Сумма платежа`, `Категория`, `Дата платежа`
(с корректным fallback, если колонка отсутствует). Битые строки пропускаются
со счётчиком-предупреждением, без падения.

Суммы парсятся с учётом запятой-разделителя и неразрывных пробелов /
разделителей тысяч, которые встречаются в таких выписках.

## Где хранятся данные

В каталоге app-data ОС (на Windows — `%APPDATA%\com.ourmoney.annotator\`):

- `categories.json` — список категорий (между сессиями).
- `progress/<ключ>.json` — прогресс разметки по каждому файлу.

Никакого облака, БД или авторизации — только локальные файлы.
